### Fullstack Engineer Fact-Checking Standards

**Philosophy**: Full-stack framework capabilities and performance characteristics must be verified across both frontend and backend concerns.

---

#### Critical Claim Types

**1. Full-Stack Framework Capabilities** (Required Quality: B+, 85%+)
- Next.js, Nuxt, SvelteKit features, SSR/CSR/SSG support

**Verification Requirements**: Official framework documentation, release notes, feature matrices

**2. SSR/CSR Performance** (Required Quality: B+, 85%+)
- Rendering performance, hydration costs, Time to Interactive

**Verification Requirements**: Official benchmarks, web.dev metrics, Lighthouse scores from documented sources

**3. API Gateway Patterns** (Required Quality: B+, 85%+)
- BFF patterns, API composition, caching strategies

**Verification Requirements**: Official API gateway docs (Kong, Traefik, etc.), architecture patterns documentation

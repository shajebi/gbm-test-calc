### Backend Engineer Fact-Checking Standards

**Philosophy**: Technical specifications and performance benchmarks must be verified with official sources and established benchmarks.

---

#### Critical Claim Types

**1. Framework Performance** (Required Quality: B+, 85%+)
- Throughput metrics
- Latency benchmarks
- Scalability limits

**Verification Requirements**:
- Official benchmarks (TechEmpower, TPC-C)
- Vendor documentation
- Peer-reviewed performance studies

**2. API Standards** (Required Quality: B+, 85%+)
- OpenAPI/Swagger specifications
- REST/GraphQL best practices
- Protocol versions

**Verification Requirements**:
- Official specification documents (OpenAPI, GraphQL Foundation)
- RFC documents for protocols
- W3C/IETF standards

**3. Database Benchmarks** (Required Quality: B+, 85%+)
- Query performance
- Connection limits
- Storage capabilities

**Verification Requirements**:
- Official database documentation
- TPC benchmarks
- Vendor performance guides
